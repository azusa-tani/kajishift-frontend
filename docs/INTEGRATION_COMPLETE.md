## 🔧 設定が必要な項目

### 1. CORS設定

バックエンドのCORS設定を確認してください。

**現在の設定** (`kajishift-backend/src/index.js`):
```javascript
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'http://localhost:5500',
  credentials: true
}));
```

**フロントエンドのポートに合わせて設定**:
- フロントエンドが `http://localhost:5500` で動く場合: 設定不要（デフォルト）
- フロントエンドが別のポートで動く場合: `.env` ファイルに `CORS_ORIGIN` を設定

**例** (`.env` ファイル):
```env
CORS_ORIGIN=http://localhost:5500
# または複数のオリジンを許可する場合
# CORS_ORIGIN=http://localhost:5500,http://localhost:3001
```

### 2. バックエンドサーバーの起動

バックエンドサーバーが起動していることを確認してください：

```bash
cd kajishift-backend
npm run dev
```

サーバーは `http://localhost:3000` で起動します。

### 3. フロントエンドの起動

フロントエンドを起動してください：

- **Live Server** (VS Code拡張機能) を使用する場合: `http://localhost:5500`
- **その他のサーバー** を使用する場合: 対応するポートで起動

## 📝 使用方法

### 1. ログイン

1. ブラウザで `http://localhost:5500/customer/login.html` を開く
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック
4. ログイン成功後、ダッシュボードにリダイレクト

### 2. 登録

1. ブラウザで `http://localhost:5500/customer/register.html` を開く
2. 必要な情報を入力
3. 「登録する」ボタンをクリック
4. 登録成功後、自動的にログインしてダッシュボードにリダイレクト

### 3. ダッシュボード

- 認証が必要なページは自動的に認証チェックが行われます
- 未認証の場合はログインページにリダイレクトされます
- ユーザー情報と予約情報が自動的に取得・表示されます

## 🔍 トラブルシューティング

### CORSエラーが発生する場合

1. バックエンドの `.env` ファイルで `CORS_ORIGIN` を確認
2. フロントエンドのポートと一致しているか確認
3. バックエンドサーバーを再起動

### ログインできない場合

1. バックエンドサーバーが起動しているか確認
2. ブラウザの開発者ツール（F12）でエラーを確認
3. ネットワークタブでAPIリクエストのステータスを確認
4. バックエンドのログを確認

### トークンが無効になる場合

- トークンは `localStorage` に保存されます
- ブラウザの開発者ツールで `localStorage.getItem('token')` を確認
- トークンが無効な場合は自動的にログインページにリダイレクトされます

## 🔐 セキュリティ注意事項

1. **トークンの管理**
   - トークンは `localStorage` に保存されます
   - XSS攻撃に注意が必要です
   - 本番環境では `httpOnly` クッキーの使用を検討してください

2. **HTTPSの使用**
   - 本番環境では必ずHTTPSを使用してください
   - トークンが平文で送信されるのを防ぎます

3. **CORS設定**
   - 本番環境では適切なオリジンのみを許可してください
   - ワイルドカード（`*`）の使用は避けてください
